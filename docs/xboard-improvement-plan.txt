## 3. 分阶段改进路线（执行版）

## Phase 0（2026-02-24 ~ 2026-03-02）：安全止血与信任修复

目标：消除可被利用的安全风险，修复支付成功误导。

1. TLS 校验收口
- 动作：移除生产路径 `badCertificateCallback => true`；如确需调试，仅限 Debug 宏开关。
- 输出：安全网关策略文档 + 代码守卫。
- 验收：`rg "badCertificateCallback\\s*=\\s*\\(.*\\)\\s*=>\\s*true" lib/xboard` 仅允许测试代码出现。

2. 记住密码安全化
- 动作：不再持久化明文密码；改为 token 或系统安全存储（Keychain/Keystore）。
- 输出：迁移策略（旧字段清理、灰度回滚策略）。
- 验收：`xboard_saved_password` 不再落明文，旧版本升级后可自动清理。

3. 支付成功链路纠偏
- 动作：`_completePayment()` 改为“稍后查看订单状态”，不再提示成功；成功仅由订单状态 `completed` 驱动。
- 输出：支付状态机与文案规范。
- 验收：仅后端确认成功时出现成功提示，客服“已支付未到账”工单占比下降。

4. 日志脱敏与分级
- 动作：release 关闭 body/header 明文日志；脱敏 `Authorization`、邮箱、可能的 token 字段。
- 输出：日志策略（Debug/Release）。
- 验收：release 包日志抽样无敏感字段。

## Phase 1（2026-03-03 ~ 2026-03-16）：可靠性与体验补强

目标：降低“能用但不稳”的体感，增强弱网可恢复性。

1. 存储初始化从“静默失败”改为“显式状态”
- 动作：移除 `_PlaceholderStorage` 或只用于只读场景；写操作失败可感知并重试。
- 验收：存储未就绪时 UI 有明确状态，关键流程（登录、记住密码）可重试可观测。

2. 令牌过期处理一致化
- 动作：将 token 过期收敛到单一守卫，避免各页面重复弹窗/竞态。
- 验收：401 后统一回收会话并路由至登录；无重复弹窗。

3. 离线能力从“局部缓存”升级到“最小可用”
- 动作：定义离线读模型（用户信息、订阅摘要、上次可用套餐快照、上次订单摘要、关键通知）。
- 验收：断网时主界面可展示“最近快照 + 明确离线标识”，不出现核心白屏。

4. 刷新策略治理
- 动作：tab 切换改为 TTL + 事件驱动，避免每次都打远端。
- 验收：同等使用路径下请求数下降，冷/热切页响应时间更稳定。

## Phase 2（2026-03-17 ~ 2026-04-07）：结构化降债与质量体系

目标：把“架构优势”落实到可维护实现层。

1. 大文件拆分与边界治理
- 动作：优先拆 `vpn_hero_card.dart`、`plan_purchase_page.dart`、`notice_banner.dart`、`xboard_user_provider.dart`。
- 验收：单文件目标 `< 400` 行（允许少量例外但需有说明）；组件职责清晰。

2. 状态管理解耦
- 动作：支付、订阅、用户 UI 状态分域，避免共享 `userUIStateProvider`。
- 验收：跨页面加载/错误互相污染问题消失。

3. 错误处理策略统一
- 动作：Provider/Feature 层减少“catch 后直接返回 false/null”；明确可恢复与不可恢复错误分类。
- 验收：关键链路错误可追踪（含 error code、埋点、用户可读提示）。

4. 测试补网
- 动作：新增最小回归集：认证、支付状态机、订阅刷新、初始化降级、i18n 文案键检测。
- 验收：`lib/xboard` 关键路径测试覆盖率阶段目标 40%（先保回归，再扩展）。
